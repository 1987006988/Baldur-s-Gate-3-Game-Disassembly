## Question

`终局组织与收束压力` 能否被写成 `Act 3` 的“组织级终局压缩 / ending feedback 读取层”，即：它的核心价值不在于提供一份终局剧情提要、Boss 顺序表或结局百科，而在于说明为什么已在 `Rivington` 被过滤、在 `Wyrm's Crossing` 被门槛化、并在 `Lower City` 被并排承接的 late-game 问题束，会在这里进一步被更少、更硬的终局组织位读取、重排并压向结算？

## Research boundary

- 本 note 只处理 `终局组织与收束压力` 作为 `Act 3` 最后一个子单元的结构职责，不重写 `Rivington`、`Wyrm's Crossing`、`Lower City` 的区块级判断。
- 本 note 不写终局剧情复述、Boss 顺序表、派系百科或结局罗列，也不把某个终局场景误写成整个后期的唯一解释器。
- 本 note 只写到公开可验证的层级：城市阶段定位、选择驱动的 ending feedback、后期 flow / reaction 维护、多入口设计语言，以及 Journal 的公开结构语言。

## What can be written as facts

- `事实`：官方 About 页把 `the sprawling city of Baldur's Gate` 作为 BG3 的阶段级城市节点直接点名。来源：`BG3-OFF-001`
- `事实`：`Patch 7` 明确加入新的 evil ending cinematics，并把它们表述为会 `depending on the choices you make` 出现。来源：`BG3-OFF-002`
- `事实`：`Patch 7` 同时继续集中修复后期的 `writing and flow`、`scripting`、角色反应与长休 / 营地边界。来源：`BG3-OFF-002`
- `事实`：Adam Smith 继续用 `Swiss cheese` 描述 BG3 的多入口 / problem-solving 结构，并明确说团队上线后继续补了大量额外 dialogue / cinematics，去覆盖更高密度的状态与结局排列。来源：`BG3-OFF-003`
- `事实`：官方 Journal 文档持续把任务公开表述为 `Category -> Quest -> Objective -> Step`，并把 `trigger updates` 作为公开更新入口。来源：`BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`

## What can only be written as inferences

- `推断`：`终局组织与收束压力` 当前最稳的写法，不是“终于来到最后剧情段落”，而是 `Act 3` 的组织级终局压缩层：它把已在 `Lower City` 并排承接的问题束，继续压成更少、更硬的终局组织位与 ending feedback 读取压力。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：与 `Lower City` 的问题不同，`终局组织与收束压力` 不再主要追问“哪些问题束必须开始并排处理”，而是在追问“哪些已并排处理的问题束现在必须被终局组织位读取、重排并逼向结算”；它因此更接近组织级压缩，而不是继续增加城市密度本身。基于 `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：与常见的“终局只剩收线”印象不同，BG3 在这里更像让 `Swiss cheese` 式多入口进入更高负载的终局组织层：入口并未自动消失，但每个入口都更容易连到组织级压缩、角色反应读回与 ending feedback。基于 `BG3-OFF-002`, `BG3-OFF-003`
- `推断`：因此这块更适合被回写到 `01_macro_structure.md`、`02_quests_and_choices.md` 与 `05_implementation_validation.md`，而不是先去写终局剧情摘要；因为当前最稳的增量是“组织级压缩如何成立”，不是“某场终局事件的细节目录”。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`

## What must remain open questions

- `待验证问题`：哪些 `Lower City` 级别的高密度并行束已经足以升级为终局组织级压缩判断，哪些仍更适合留在城市核心并行 / 局部结算层？
- `待验证问题`：除 evil ending cinematics 与后期 flow / scripting 维护外，还有哪些同级别的官方条目足以更直接支撑“组织级终局压缩”这一判断？
- `待验证问题`：终局组织与收束压力中的哪些读回已足以稳定压到 `objective / step` 层，哪些目前仍更适合停留在“organization bundle + ending feedback”这一公开边界？

## Notes

- 当前最稳的仓库口径应是：`Act 3` 先证明“后期为什么会同时更密、更近结算”，`Rivington` 证明“这层高密度如何第一次被过滤成入口顺序”，`Wyrm's Crossing` 证明“这些问题束如何被桥梁 / 门槛式第二层重排”，`Lower City` 证明“这些已重排的问题束如何在城市核心并排承接”，而 `终局组织与收束压力` 则继续证明“这些并排承接的问题束如何被压向组织级收束与 ending feedback”。
- 这份 note 的作用不是替代终局剧情梳理，而是给阶段 4 / `Act 3` 提供一个干净出口：现在可以闭合 `Act 3`，而不是把所有终局信息继续回填到 `Lower City`。
