- `事实`：`BG3-OFF-007` 直接确认营地承担合作模式队伍成员去留处理，并补入了 `Karlach` 在跨 `Act` 间隙的反思时刻。来源：`BG3-OFF-007`

- `事实`：现有公开来源已经足以证明营地反馈链并非只围绕 `Dark Urge bard` 与 `Minthara` 两个高辨识度个案被维护；维护对象至少还扩展到 `Karlach`、Astarion、Voss、Mizora 与 Wyll。来源：`BG3-OFF-007`, `BG3-OFF-010`

## What can only be written as inferences

- `推断`：`BG3-OFF-007` 与 `BG3-OFF-010` 的新增价值主要是补强“营地反馈链的维护广度”，而不是生成第二条公开营地主干。
- `推断`：在当前公开来源下，阶段 5 / `Companion / Camp / Long Rest` 的最稳结构已经收束为“1 条 public camp fold spine + supporting bundles + 1 个 late-game handoff boundary”，而不是“2 条对称营地主干”。
- `推断`：因此营地模块下一步不应继续搜集零散 camp scene 条目，而应把 ceiling 结论沉入实现验证层，作为阶段 5 出口判断的一部分。
