# Case Note

## Case name

Combat / Environment 跨阶段边界：second spine ceiling / late-game boundary

## Related module

`docs/03_analysis/04_combat_and_environment.md`

次级回流模块：
- `docs/03_analysis/05_implementation_validation.md`

## Question

在当前公开来源下，`Shadow-Cursed Lands` 之后的 late `Act 2` / `Act 3` 锚点，能否被稳定压成阶段 5 / `Combat / Environment` 的第二条 cross-stage combat spine；还是更稳地只能停在 `convergence`、`entry filter`、`bridge / gate`、`parallel resolution` 与 `organization / ending feedback` 这些 supporting bundle 层？

## Research boundary

- 本 note 只处理 `Shadow-Cursed Lands` 之后的 late `Act 2` / `Act 3` 锚点，不回头重写首条 “early local agency -> regional pressure loop” 环境主干。
- 本 note 不把 `Last Light / Moonrise / Gauntlet` 或 `Act 3` 城市子包重新摊成 encounter / 机关 / cheese 列表。
- 本 note 不把 quest / camp 的既有 bundle 直接改写成 combat spine；这里只判断它们在战斗 / 环境层最多能升级到哪里。

## What can be written as facts

- `事实`：官方 About 页把 `shadow-cursed forests` 与 `the sprawling city of Baldur's Gate` 都作为 BG3 差异化阶段节点直接点名。来源：`BG3-OFF-001`
- `事实`：`Patch 7` 明确集中修复 `Act II` / late-game 的 `Writing and Flow`、`Scripting`、角色反应、长休 / 营地边界，并新增会根据玩家选择出现的 evil ending cinematics。来源：`BG3-OFF-002`
- `事实`：Adam Smith 用 `Swiss cheese` 描述 BG3 的多入口 / problem-solving 结构，并明确说团队上线后继续补了大量额外 dialogue / cinematics，去覆盖更高密度的状态与结局排列。来源：`BG3-OFF-003`
- `事实`：官方 Journal 文档持续把任务公开表述为 `Category -> Quest -> Objective -> Step`，并把 `trigger updates` 作为更新入口。来源：`BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`

## What can only be written as inferences

- `推断`：late `Act 2` / `Act 3` 当前并非“没有 combat / environment 价值”，而是它们的公开价值更稳地表现为 `convergence`、`filter`、`gate`、`parallel resolution` 与 `organization / ending feedback` 这些 downstream supporting bundles，而不是第二条对称环境 spine。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：与 `Grymforge` 的成熟 `combat-agency compression pack`、`Shadow-Cursed Lands` 的 `regional pressure loop` 不同，late-game 公开来源更集中地暴露“状态如何被压缩、过滤、重排和读回”，而不是“新的环境 agency 如何稳定组织整段推进”。基于 `BG3-OFF-002`, `BG3-OFF-003`
- `推断`：因此阶段 5 / `Combat / Environment` 当前最稳的写法，不是强行为 late `Act 2` / `Act 3` 凑第二条对称 spine，而是把首条环境主干保留为正式 spine，再把这些 late-game 锚点降回 first spine 的 downstream supporting bundles。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`

## What must remain open questions

- `待验证问题`：未来若要把任何 late-game 锚点重新升级到第二条环境 spine，最少需要什么级别的公开来源？当前最缺的是直接点名 late-game `complex combat encounters`、区域级风险循环，或更明确的环境 problem-solving 描述。
- `待验证问题`：`Act 3` 现有城市子包里，是否存在尚未纳入仓库的更强官方环境来源，能把其中某一块从 `filter / gate / parallel resolution bundle` 再向上提升？
- `待验证问题`：`Last Light / Moonrise / Gauntlet` 是否还能在后续来源中暴露出更直接的 combat / environment 锚点，而不只是 `Act II` 的收束与读回职责？

## Notes

- 当前最稳的仓库口径应是：首条环境主干成立，但 second spine ceiling 也已成立。
- 这不等于 late-game 在战斗 / 环境层失效；它只意味着当前公开来源对 late-game 的最稳支持方式，是 supporting bundles，而不是第二条对称环境 spine。
