## Question

`Lower City` 能否被写成 `Act 3` 的“更高密度的内城并行 / 结算区”，即：它的核心价值不在于把内城再拆成更多地点与派系清单，而在于说明为什么已经在 `Rivington` 被第一次过滤、在 `Wyrm's Crossing` 被第二层门槛化重排的 late-game 问题束，会在这里开始同时以更高密度被承接、并行推进、局部读回与逼近结算？

## Research boundary

- 本 note 只处理 `Lower City` 作为 `Act 3` 第三个城市子块的结构职责，不展开 `终局组织与收束压力` 的终局组织包。
- 本 note 不写内城景点导览、派系列表、支线总表，也不把 `Lower City` 直接升级成整个后期的唯一结算核。
- 本 note 只写到公开可验证的层级：城市阶段定位、选择驱动的 ending feedback、开发者对多入口与高密度状态排列的表述，以及 Journal 的公开结构语言。

## What can be written as facts

- `事实`：官方 About 页把 `the sprawling city of Baldur's Gate` 作为 BG3 的阶段级城市节点直接点名。来源：`BG3-OFF-001`
- `事实`：`Patch 7` 明确加入新的 evil ending cinematics，并把它们表述为会 `depending on the choices you make` 出现。来源：`BG3-OFF-002`
- `事实`：`Patch 7` 同时继续集中修复后期的 `writing and flow`、`scripting`、角色反应与长休 / 营地边界。来源：`BG3-OFF-002`
- `事实`：Adam Smith 继续用 `Swiss cheese` 描述 BG3 的多入口 / problem-solving 结构，并明确说团队上线后继续补了大量额外 dialogue / cinematics，去覆盖更高密度的状态与结局排列。来源：`BG3-OFF-003`
- `事实`：官方 Journal 文档持续把任务公开表述为 `Category -> Quest -> Objective -> Step`，并把 `trigger updates` 作为公开更新入口。来源：`BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`

## What can only be written as inferences

- `推断`：`Lower City` 当前最稳的写法，不是“终于进内城后内容更多”，而是 `Act 3` 的更高密度内城并行 / 结算区：它把已在 `Rivington` 被过滤、在 `Wyrm's Crossing` 被门槛化的问题束，继续压成同时承接、重读与局部结算的内城负载。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：与 `Wyrm's Crossing` 的“哪些问题已经变成必须跨过的门槛”不同，`Lower City` 更像在追问“哪些已跨过门槛的问题束现在必须开始并排被处理、读回与逼近结算”；它因此更接近高密度承压与局部结算，而不是再次扮演桥梁 / 检查点。基于 `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：与后续 `终局组织与收束压力` 相比，`Lower City` 仍不应被写成整个后期唯一的终局核；它更像在城市核心把 late-game bundle 压到“高密度并行 + 逼近结算”的阶段，而最终的组织级压缩还应留给下一子块。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`

## What must remain open questions

- `待验证问题`：哪些在 `Wyrm's Crossing` 被压成桥梁 / 门槛重排的 late-game bundle，到 `Lower City` 已经足以写成更高密度的并行 / 结算负载，哪些其实更适合继续留到 `终局组织与收束压力`？
- `待验证问题`：哪些 `Lower City` 级别的阅读顺序变化已经足以稳定压到 `objective / step` 层，哪些目前仍更适合停留在“高密度 quest bundle + ending feedback”这一公开边界？
- `待验证问题`：除 `Patch 7` 的 ending feedback 与后期 flow / scripting 维护外，还有哪些同级别官方条目足以更直接支撑“`Lower City` 承担更高密度并行 / 结算区职责”这一判断？

## Notes

- 当前最稳的仓库口径应是：`Act 3` 先证明“后期为什么会同时更密、更接近结算”，`Rivington` 证明“这层高密度如何第一次被过滤成入口顺序”，`Wyrm's Crossing` 证明“这些问题束如何进一步变成桥梁 / 门槛式第二层重排”，而 `Lower City` 则继续证明“这些已被重排的问题束如何在城市核心开始以更高密度并行承接与逼近结算”。
- 这份 note 的作用不是替代后续 `终局组织与收束压力`，而是先给它提供干净边界：高密度内城承压已经发生，但终局组织级压缩仍未在本轮提前写死。
