# Case Note

## Case name

救出 Halsin 后的营地同伴对话恢复

## Related module

`docs/03_analysis/03_party_and_camp.md`

## Question

这个案例能否作为第三类营地反馈案例，说明 BG3 会把“任务推进之后，营地中的同伴反馈是否可达”当作正式维护的系统边界？

## What the player sees

- 玩家完成前置推进：救出 Halsin。
- 玩家回到营地后，预期同伴会重新可对话，并对前序推进给出后续反馈。
- 如果这些营地对话无法触发，玩家会感到任务推进没有被营地层接住。

## System reading

- 这个案例的关键不在于 Halsin 个人，而在于“任务推进之后，营地中的 companion dialogue 可达性”会被官方单独修复。
- 它说明营地反馈链至少包含一个低层边界：任务推进完成后，营地中的同伴回应必须重新开放。
- 相比 `Dark Urge bard` 与 `Minthara`，这条案例更偏“反馈可达性恢复”，而不是“完整剧情折返链”。

## Evidence status

- `事实`：
  - `BG3-OFF-009` 直接确认：救出 Halsin 后，玩家可以再次在营地与同伴发起对话。
  - `BG3-OFF-009` 直接把问题描述成营地同伴对话与反馈缺失。
- `推断`：
  - 营地中的同伴反馈可达性至少在一个非 `Dark Urge`、非 `Minthara` 案例上是正式维护的边界。
  - 这条案例可以作为第三类营地反馈候选，用来证明当前结构不是只由两个高辨识度特例支撑。
- `待验证问题`：
  - 这条候选是否足以进入首轮正文？当前判断是否定的，因为来源仍偏单条、且来自 EA 热修。
  - 是否还能补到正式版或更高层来源，把它从“候选”提升成“正文可用案例”？
