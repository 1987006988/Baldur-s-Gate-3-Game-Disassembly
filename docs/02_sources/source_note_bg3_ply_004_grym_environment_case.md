## Title

Reddit: We failed the "Owlbear from the top rope" strat.

## Type

community case

## Why this source matters

这条社区案例不用于证明系统事实本身，而用于锁定一个值得继续补强的案例方向：`Grymforge / Grym` 战斗经常被玩家理解成一个“可被环境、移动路线、体型与战前布置重写”的遭遇，而不是只能按正面数值对殴处理。这正好能补足 `02_quests_and_choices` 当前案例结构过度偏向叙事反馈的问题。

## Reliable facts

- 玩家社区持续把 `Owlbear from the top rope`、远程击打机关、利用平台 / 高差 / 落体与熔炉机关处理 Grym 等方案，当作一个高辨识度的非常规解法集合来讨论。
- 该讨论同时暴露了这种解法的不稳定性：命中判定、落点、版本调整与具体执行细节都可能影响结果。
- 同一讨论中，玩家还会把“射击拉杆 / 机关”“从非常规路线进入或布置开战位置”等方式视作同一类环境重写策略。

## Useful inferences

- `推断`：Grymforge / Grym 是一个很适合补入任务模块的“战斗 / 环境绕解”案例方向，因为它把任务推进、空间移动、环境机关和战斗结果绑在一起。
- `推断`：这个案例的价值不在于某个具体 cheese 是否永远稳定，而在于它显示玩家确实把 BG3 的某些战斗遭遇理解为“任务问题 + 空间问题 + 机关问题”的复合解题空间。

## Limits and bias

- 这条来源是社区案例，只能说明玩家如何描述和利用该遭遇，不能直接当作系统事实。
- 讨论里混杂版本差异、失败经验、戏谑表达与个人执行误差。
- 它适合做“案例入口”，不适合单独承担实现层结论。
