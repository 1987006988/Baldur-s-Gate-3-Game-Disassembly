# Source Note

## Source ID

`BMK-002`

## Title

Reverse Design: Final Fantasy 6 - 1

## Type

benchmark / reverse design method essay

## Why this source matters

这条来源给了本项目一个非常重要的方法约束：先从作品已经呈现出的设计结果出发，反向追问设计决策，并尽量只说那些能被作品自身证据支撑的判断。对 BG3 来说，这正好适合拿来处理“玩家已经感受到自由和反馈，但系统解释还需要反推”的问题。

## Reliable facts

- 作者明确提出，要“reverse-engineer”经典游戏背后的设计决策与过程。
- 作者强调，研究经典游戏设计史与研究设计基础同样重要。
- 作者明确表示，自己会尽量避免讨论那些没有被作品设计本身具体证据支持的内容。

## Useful inferences

- `推断`：BG3 的总拆解逻辑应先抓住可感知现象，再反推系统逻辑，而不是先搭一套抽象理论硬套作品。
- `推断`：仓库应持续区分“当前最强解释”和“已被公开证据支持的事实”，这与 Reverse Design 的谨慎风格一致。

## Limits and bias

- 这是一条方法论文，不是 BG3 专属资料。
- 它适合给项目定拆解纪律，不适合直接提供 BG3 的具体结论。
