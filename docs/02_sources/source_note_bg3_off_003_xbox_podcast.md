# Source Note

## Source ID

`BG3-OFF-003`

## Title

Official Xbox Podcast: Baldur’s Gate 3 comes to Xbox and our holiday gift ideas

## Type

developer explanation

## Why this source matters

这是少数能直接听到 Adam Smith 解释“为什么要给玩家这么多解法、为什么关卡会做成多孔结构、为什么团队保留奇怪玩法”的一手开发者表述。它非常适合给任务与选择模块提供设计意图层的支撑。

## Reliable facts

- Adam Smith 在官方 Xbox Podcast 访谈中把 BG3 的关卡/路径设计比作 “Swiss cheese”，强调关卡有很多洞，玩家会从不同位置反复穿出。
- 他明确把这种设计与玩家的 out-of-the-box problem-solving 联系起来，并举了月出之塔 owlbear 跳砸等玩家发现的非常规解法。
- 他提到团队在上线后为了兑现既有承诺，补了大量额外对话和过场，以覆盖更高密度的结局与状态排列。
- 访谈中还提到团队愿意保留一些“很怪但很有趣”的玩法，而不是把所有非预期但有趣的行为都压平。

## Useful inferences

- `推断`：任务与选择回流之所以成立，不只靠对白分支，还依赖多入口关卡和允许玩家偏离“标准解法”的场景结构。
- `推断`：Larian 的设计目标不是给出唯一正确路线，而是给玩家留下可穿梭的“孔洞”，让任务推进、潜入、对话、战斗在同一区域内互相转换。

## Limits and bias

- 开发者访谈说明的是设计意图和方法，不等同于每个具体任务都同样实现充分。
- 访谈中的案例偏代表性展示，容易偏向成功、好玩的例子。
- 这条来源更适合支撑“设计哲学”，不适合作为单一任务后果的直接事实依据。
