# Case Note

## Case name

Grymforge / Grym 环境解法路径

## Related module

`docs/03_analysis/02_quests_and_choices.md`

## Question

能否用 Grymforge / Grym 这一类遭遇，补足 `02_quests_and_choices` 当前案例结构过于偏向叙事反馈的问题，说明 BG3 的“任务回应”也可能来自战斗与环境层面的非常规解法？

## 为什么锁定这条案例

- 它能把“选择”从对白与营地事件，扩展到战前布置、路线选择、环境利用与机关调用。
- 它和 `04_combat_and_environment` 天然相连，适合做模块之间的桥梁案例。
- 它能说明：玩家觉得“世界承认了我的做法”，有时并不是因为 NPC 说了什么，而是因为系统允许你用非常规方式完成原本看似固定的遭遇。

## 当前可确认的案例方向

- 以 `Grymforge / Grym` 为中心，观察玩家如何通过高差、落体、机关、远程击打拉杆、非常规站位与战前准备来重写战斗解决方式。

## 初步证据判断

### 可写为事实

- `BG3-OFF-003` 已提供高层设计意图：Adam Smith 明确把 BG3 的关卡设计比作 `Swiss cheese`，强调多孔结构与非常规 problem-solving。
- `BG3-OFF-006` 已提供高层区域框架：官方把 Grymforge 明确描述为包含 `new questlines, choices, ... complex combat encounters` 的区域，并在同一更新中引入新 Weapon Actions 与更多战术能动性。
- `BG3-PLY-004` 可作为案例入口，证明玩家确实把 `Grymforge / Grym` 遭遇理解为一个可以通过环境与空间重写的解题场景；但这只能写成玩家案例事实，不能写成系统普遍事实。

### 只能写为推断

- `Grymforge / Grym` 很可能适合作为任务模块中的“战斗 / 环境绕解”补充案例，因为它把遭遇完成方式与空间利用绑定在一起。
- 这类案例能够补足 `02_quests_and_choices` 当前偏重叙事反馈的结构偏差。
- 但在当前证据结构下，它更适合先作为 `04_combat_and_environment` 的主深挖案例，再反向支撑任务模块。

### 必须保留为待验证问题

- 是否有更直接的官方来源支持这条案例，而不只停留在开发者高层表述、区域框架与社区案例之间。
- 这条案例应最终落在任务模块、战斗模块，还是同时作为桥梁案例出现在两边。
- 哪一条具体执行路径最适合进入首轮正文，而不是只停留在“案例方向”层。
