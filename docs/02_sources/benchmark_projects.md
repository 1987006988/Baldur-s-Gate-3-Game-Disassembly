# Benchmark Projects

本页不是收链接仓库，而是记录“值得借鉴的拆解项目类型”，并明确哪些部分适合 BG3，哪些不适合照搬。

## 当前已压实的对照方法

### `BMK-002` Reverse Design

- 借鉴点：从玩家实际体验到作品内部结构，再到可被作品自身证据支持的设计决策；强调不要脱离对象乱猜。
- 适合 BG3 的原因：BG3 的高反应性首先表现为玩家体验到的现象，必须先抓现象，再反推系统与实现验证入口。
- 不该照搬的地方：BG3 是多系统耦合作品，不能只做单模块长文逆向分析。

### `BMK-003` Boss Keys

- 借鉴点：用图、门槛、依赖、入口与回环来分析非线性空间。
- 适合 BG3 的原因：高密度区域包、多入口推进、门槛与绕路，非常适合借这种图式思维分析。
- 不该照搬的地方：BG3 不只是空间依赖图，还要解释任务状态、营地折叠、同伴反馈与 Act 收束。

## 仍需要寻找的对照类型

- 更强的“案例驱动 + 系统耦合”拆解例子
- 更强的“任务状态 / 反应性 / 实现验证”混合写法
- 能把空间、任务和反馈折叠成同一张图的对照方法

## 当前用途

- 为 `deconstruction_display_overview.md` 提供方法论参照
- 为 `deconstruction_master_execution_plan.md` 和 `deconstruction_granular_codex_plan.md` 提供可执行拆解范式
- 帮助判断“什么该借，什么不该照搬”
