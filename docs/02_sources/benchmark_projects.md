# Benchmark Projects

本页不是收链接仓库，而是记录“值得借鉴的拆解项目类型”。

## 需要寻找的对照类型

- 机制拆解型：把体验现象拆到规则层与资源循环
- 叙事系统缝合型：解释剧情表达如何借系统成立
- 关卡与空间结构型：解释区域、入口、回流与推进
- 工业化验证型：会引用访谈、补丁、工具链或 UI 证据

## 记录框架

后续补具体案例时，至少记录:

- 项目名称
- 类型
- 值得借鉴的结构
- 不该照搬的地方
- 对本仓库的启发

## 当前用途

- 为 `module_template.md` 与 `03_analysis/*` 提供结构参照
- 帮助判断“什么叫可执行拆解”，避免滑向感想文
