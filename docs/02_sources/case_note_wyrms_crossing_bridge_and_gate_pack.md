## Question

`Wyrm's Crossing` 能否被写成 `Act 3` 的“桥梁 / 门槛 / 第二层结构重排”区域包，即：它的核心价值不在于补一段过桥路线或检查点导览，而在于把已经在 `Rivington` 被第一次分束的后期并行线，继续压成更靠近城市核心、带有门槛感与重排压力的第二层阅读结构？

## Research boundary

- 本 note 只处理 `Wyrm's Crossing` 作为 `Act 3` 第二个城市子块的结构职责，不展开 `Lower City` 或终局组织的地点级子包。
- 本 note 不写桥头地点导览、派系列表、检查点清单，也不把 `Wyrm's Crossing` 直接升级成高密度结算核。
- 本 note 只写到公开可验证的层级：城市阶段定位、选择驱动的后期读回、开发者对多入口与高密度状态排列的表述，以及 Journal 的公开结构语言。

## Bridge-and-gate frame

| 维度 | 当前能写成事实的部分 | 当前最稳推断 | 仍待验证 |
| --- | --- | --- | --- |
| 空间组织问题 | 官方 About 页把 `the sprawling city of Baldur's Gate` 作为差异化阶段级城市节点直接点名。来源：`BG3-OFF-001` | 既然 `Act 3` 被公开表述为城市化阶段，那么在 `Rivington` 已经完成第一层入口过滤之后，`Wyrm's Crossing` 当前最稳的写法就不是“再来一块城市前厅”，而是把外环问题推进到更接近城市核心的桥梁 / 门槛节点。 | 公开来源仍不足以直接点名 `Wyrm's Crossing` 内部每条并行线各自承担的空间功能。 |
| 任务 / 派系交错问题 | `Patch 7` 一方面把新的 ending feedback 写成会根据玩家选择出现，另一方面继续集中修复后期 `writing and flow`、`scripting`、角色反应与长休 / 营地边界。来源：`BG3-OFF-002` | 这说明 `Act 3` 的任务压力不是“有没有后果”，而是“更多后果与读回更快挤到玩家面前”；因此 `Wyrm's Crossing` 更适合作为第二层重排包，把已被 `Rivington` 分束的问题进一步改写成带门槛感的阅读顺序，而不是立即结算它们。 | 目前仍缺更直接的公开条目，能把哪些 late-game bundle 在这里变成门槛、哪些继续延后到 `Lower City` 写成事实链。 |
| 状态收束问题 | Adam Smith 继续用 `Swiss cheese` 描述 BG3 的多入口结构，并明确说团队上线后继续补了大量额外 dialogue / cinematics，去覆盖更高密度的状态与结局排列。来源：`BG3-OFF-003` | `Wyrm's Crossing` 当前最稳的价值，不是取消多入口，而是让多入口在更高收束压力下继续存在，但每一次跨越都更像对后续状态负载的再分配；它比 `Rivington` 更靠近门槛，比 `Lower City` 更少承担最终结算。 | 还不能把这条高层设计语言直接外推成 `Wyrm's Crossing` 内某个固定通行规则或具体状态机。 |
| 公开验证边界 | 官方 Journal 文档持续把任务公开表述为 `Category -> Quest -> Objective -> Step`，并把 `trigger updates` 作为公开更新入口。来源：`BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014` | 因此本轮对 `Wyrm's Crossing` 最稳的任务层写法，是“第二层 bridge / gate bundle 如何重排 late-game quest reading order”，而不是假装已经掌握这块内部的 objective / step 编排。 | 仍缺能够把 `Wyrm's Crossing` 的哪些门槛已经足以压到 objective / step 层的公开材料。 |

## What can be written as facts

- `事实`：官方 About 页把 `the sprawling city of Baldur's Gate` 作为 BG3 的阶段级城市节点直接点名。来源：`BG3-OFF-001`
- `事实`：`Patch 7` 明确加入新的 evil ending cinematics，并把它们表述为会 `depending on the choices you make` 出现。来源：`BG3-OFF-002`
- `事实`：`Patch 7` 同时继续集中修复后期 `writing and flow`、`scripting`、角色反应与长休 / 营地边界。来源：`BG3-OFF-002`
- `事实`：Adam Smith 继续用 `Swiss cheese` 描述 BG3 的多入口 / problem-solving 结构，并明确说团队上线后继续补了大量额外 dialogue / cinematics，去覆盖更高密度的状态与结局排列。来源：`BG3-OFF-003`
- `事实`：官方 Journal 文档持续把任务公开表述为 `Category -> Quest -> Objective -> Step`，并把 `trigger updates` 作为公开更新入口。来源：`BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`

## What can only be written as inferences

- `推断`：`Wyrm's Crossing` 当前最稳的写法，不是“桥头通道段”或“再次筛选入口的前厅”，而是 `Act 3` 的第二层结构重排区：它把已经在 `Rivington` 被第一次分束的后期并行线，继续压成更靠近城市核心的桥梁 / 门槛阅读结构。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：与 `Rivington` 的“先碰哪一束问题”不同，`Wyrm's Crossing` 更像在追问“哪些问题已经变成必须跨过的门槛、哪些仍可延后到内城继续展开”；它因此更接近重排，而不是第一次过滤。基于 `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：与 `Lower City` 预期承受的更高密度并行 / 结算负载相比，`Wyrm's Crossing` 更适合被写成过渡性的第二层承压节点：后期读回已经更近，但还没有被压到全量内城结算核。基于 `BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`

## What must remain open questions

- `待验证问题`：哪些在 `Rivington` 被第一次过滤的 late-game bundle，到了 `Wyrm's Crossing` 已经足以被理解成桥梁 / 门槛级重排，哪些其实更接近 `Lower City` 的高密度并行与结算负载？
- `待验证问题`：哪些 `Wyrm's Crossing` 级别的阅读顺序变化，已经足以稳定压到 `objective / step` 层，哪些目前仍更适合停留在“第二层 quest bundle 重排”这一公开边界？
- `待验证问题`：除 `Patch 7` 的 ending feedback 与 flow / scripting 维护边界外，还有哪些同级别官方条目足以更直接支撑“`Wyrm's Crossing` 负责桥梁 / 门槛重排”这一判断？

## Notes

- 当前最稳的仓库口径应是：`Act 3` 先证明“后期为什么会同时更密、更接近结算”，`Rivington` 再证明“这层高密度如何第一次被分束成入口过滤”，而 `Wyrm's Crossing` 则继续证明“这些已被分束的问题如何进一步变成桥梁 / 门槛式的第二层重排”。
- 这份 note 的作用不是替代后续 `Lower City` 或终局组织，而是先给它们提供干净的上游边界：第一层过滤已经发生，第二层门槛重排也已建立，后面的子块不需要再重复扮演桥头过滤器。
