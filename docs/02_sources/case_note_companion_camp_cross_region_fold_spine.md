# Case Note

## Case name

Companion / Camp / Long Rest：跨区域 `camp fold / delayed feedback` 主干

## Related module

`docs/03_analysis/03_party_and_camp.md`

## Question

阶段 5 能否把既有营地案例、`Act 1`/`Act 2`/`Act 3` 已完成区域包与官方补丁锚点压成第一条可复述的 `camp fold / delayed feedback` spine，同时明确哪些读出只能停在 `feedback fold / dialogue accessibility bundle` 或 `ending feedback handoff` 层，而不是新的 `objective` 主链？

## Research boundary

- 本 note 只处理“营地主干如何跨区域成立”，不扩写同伴人物志、恋爱系统或全量 camp scene 列表。
- 本 note 允许把 `Act 2` 收束点组与 `Act 3` 终局组织写入同一条链，但只把它们当作 `convergence-adjacent reread` 与 `ending feedback handoff`，不假装这些节点已经被公开证明为对称的营地场景主链。
- 本 note 复用既有区域包与营地 case note 的结论，不再重做一轮局部 patch / hotfix 侦察。

## Supporting bundles that do not enter the first spine

- `BG3-OFF-007` 暴露了 `Withers' Wardrobe` 与 Karlach 在跨 `Act` 间隙的反思时刻。
- `BG3-OFF-010` 暴露了 Astarion、Voss、Mizora、Wyll 等非 `Minthara` 的 camp scene / dialogue accessibility / 在场识别修复。
- `推断`：这两组条目当前更适合停在 `reflection / dialogue accessibility supporting bundle` 层；它们能证明营地反馈链被系统性维护，但还不足以单独撑起第二条公开跨区域 spine。来源：`BG3-OFF-007`, `BG3-OFF-010`

## What can be written as facts

- `事实`：`Dark Urge bard` 事件至少已经被官方压实到“营地临时入队的状态变化”，且 `Patch 7` 把它明确回指到 `Alfira` 与 `temporarily`。来源：`BG3-OFF-005`, `BG3-OFF-002`
- `事实`：`Grove / Goblin` 的一部分后果会以营地对话可达性与 `companion reaction` 的形式被重新读取。来源：`BG3-OFF-008`
- `事实`：`Mountain Pass / Creche` 的一部分伙伴后果会以 `camp night` 的延迟反思形式被重新读取。来源：`BG3-OFF-016`
- `事实`：`Act II` 的中盘 flow / reaction 维护与公开 Journal 结构语言足以支持“收束点组附近存在 reread / trigger-update 压缩”这一层判断。来源：`BG3-OFF-002`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `事实`：late-game `ending feedback` 会根据玩家选择出现，并与后期 `writing and flow` / `scripting` 维护共同构成公开可见的读回边界。来源：`BG3-OFF-002`

## What can only be written as inferences

- `推断`：阶段 5 现在已经拥有第一条可复述的营地主干，但它不是第二条 `Quest Reactivity`；它的结构是“前史密度生成 -> 局部 camp-state shift -> delayed camp readout -> camp-night reread -> convergence-adjacent reread -> ending-feedback handoff”。
- `推断`：真正进入第一条营地主干中心的，不是所有同伴对白，而是那些被公开来源直接点名为会在营地重新读出、重新排序或被持续维护的状态边界。
- `推断`：`Karlach` 反思时刻与非 `Minthara` camp fix 当前更适合作为“证明营地反馈链被系统性维护”的 supporting bundles，而不是对称的第二条公开 spine。来源：`BG3-OFF-007`, `BG3-OFF-010`

## What must remain open questions

- `待验证问题`：除 `Dark Urge bard`、`Minthara` 与 `Lae'zel camp night` 外，现有公开来源是否还能压出第二条不依赖高辨识度特例的公开 camp spine？
- `待验证问题`：`Act II` 收束点组中，哪些后果会稳定折返到 camp scene / camp dialogue，哪些仍更适合停在 `convergence-adjacent reread` 层？
- `待验证问题`：late-game `ending feedback` 中，哪些读回仍与营地折叠直接相连，哪些已经完全移交给组织级结算节点？
