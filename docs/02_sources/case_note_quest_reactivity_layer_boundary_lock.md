## Question

在已完成上游前置链、中盘接缝、首条 `readback spine` 与“第二条公开跨阶段链证据上限”判断之后，当前仓库里哪些 `Quest Reactivity` 锚点还能继续逼近 `objective / step`，哪些必须明确停在 `gate / tension`、`camp fold`、`ending feedback / organization bundle`？

## Research boundary

- 本 note 不新增区域包，不回到 `Act 1`、`Act 2`、`Act 3` 的地点复述。
- 本 note 不重新侦察新来源；只重排仓库中已蒸馏的官方 source note 与已完成 case note。
- 本 note 不把 `party_and_camp` 或 late-game ending feedback 强抬成第二条对称主链；这里只锁定它们的公开层级上限。
- 本 note 只写到公开可验证边界，不反推私有 objective / step 编排。

## What can be written as facts

- `事实`：`Nautiloid`、`Grove / Goblin` 与 `Mountain Pass / Creche` 都已有直接维护条目，能把少数局部边界压到 `objective / step` 邻近层。来源：`BG3-OFF-002`, `BG3-OFF-004`, `BG3-OFF-008`, `BG3-OFF-015`, `BG3-OFF-016`
- `事实`：`Shadow-Cursed Lands`、`Last Light / Moonrise / Gauntlet`、`Rivington`、`Wyrm's Crossing`、`Lower City` 与 `终局组织与收束压力` 目前的公开强度，更稳定地支持 bundle 级判断，而不是私有 objective 表。来源：`BG3-OFF-001`, `BG3-OFF-002`, `BG3-OFF-003`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `事实`：非 `Minthara` 的营地折叠与对话可达性维护真实存在，但公开条目呈现为多条 `camp fold / dialogue accessibility` 维护，而非一条具名跨 Act 主链。来源：`BG3-OFF-007`, `BG3-OFF-010`

## What can only be written as inferences

- `推断`：阶段 5 现在最稳的写法，不是“继续扩第二条主链”，而是承认只有少数局部锚点接近 `objective / step`，其余大多数结论都必须明确停在 bundle 层。基于 `BG3-OFF-002`, `BG3-OFF-007`, `BG3-OFF-010`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`, `BG3-OFF-016`
- `推断`：这意味着当前仓库对 `Quest Reactivity` 的最高公开口径应是“一条显式 spine + 少数 objective-adjacent 局部锚点 + 多个 supporting bundles”，而不是“已经拥有全局 objective 表”。基于 `BG3-OFF-002`, `BG3-OFF-004`, `BG3-OFF-008`, `BG3-OFF-012`, `BG3-OFF-013`, `BG3-OFF-014`
- `推断`：`camp fold` 与 `ending feedback / organization bundle` 的价值，主要在于约束仓库不要越级写死，而不是继续承担“再生一条主链”的任务。基于 `BG3-OFF-007`, `BG3-OFF-010`, `BG3-OFF-002`, `BG3-OFF-003`

## What must remain open questions

- `待验证问题`：`Grove / Goblin` 除 `Minthara` 外，还有哪些处理差异能被同级别官方来源压到 objective-adjacent 层？
- `待验证问题`：`Mountain Pass / Creche` 的哪些局部门槛还能继续被 Journal 结构语言压细，而不是长期停留在 gate 摘要？
- `待验证问题`：如果没有更强官方来源，阶段 5 是否应在完成 objective-adjacent 窄化验证后停止扩链，并切换到其余横向主干？
