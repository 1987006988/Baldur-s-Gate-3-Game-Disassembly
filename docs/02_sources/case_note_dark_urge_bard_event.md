# Case Note

## Case name

Dark Urge 营地 bard 事件

## Related module

`docs/03_analysis/02_quests_and_choices.md`
`docs/03_analysis/03_party_and_camp.md`

## Question

这个案例能否说明 BG3 的任务/叙事反馈不只发生在任务界面内，而会借由长休与营地事件把先前选择重新解释为角色状态与后续世界差异？

## 最小证据链骨架

### 选择点

- 玩家以 Dark Urge 身份推进流程，并在与营地 bard 相关的夜晚事件中做出关键处理；当前首轮先把“是否成功保住默认 bard”视为案例入口。

### 状态变化

- `事实`：`BG3-OFF-005` 直接表明，扮演 Dark Urge 时，某位 bard 会在营地以可控制角色身份暂时加入队伍。
- `事实`：`BG3-OFF-002` 进一步把这条变化压实到 Patch 7 正文中：官方再次回指该事件，明确这不是新同伴路线，并直接写出 `Alfira` 会暂时加入队伍、可升级、但不能使用 Magic Mirror。
- `事实`：现有首批官方来源仍不足以单独证明这条事件内部更细的分支切换细节与完整后果分布。
- `事实`：`BG3-PLY-002` 可作为社区案例线索，说明玩家普遍把这条事件理解成“长休触发 -> 角色替换或保留 -> 后续内容差异”的可感知节点；但这仍然只是案例线索。
- `推断`：如果默认 bard 可被替换、保留或引出不同后续可见差异，那么这说明营地事件并非单次演出，而是会在角色存在状态、后续对话或后续出现方式上留下痕迹。

### 后续反馈

- `事实`：`BG3-OFF-005` 与 `BG3-OFF-002` 共同强调这不是新同伴路线，而是一个短暂且带后果的状态变化。
- `事实`：目前可稳妥写成事实的，只有“玩家社区持续把这条事件讨论为一个会在长休节点改变角色后续存在与反馈方式的案例线索”，且这只能作为玩家观察，不可直接上升为全局事实。
- `推断`：对任务与选择模块而言，这条案例的价值在于它把反馈位置从“任务现场”转移到了“营地夜晚”与“后续可见差异”，因此适合和 `03_party_and_camp` 模块建立耦合。
- `待验证问题`：具体哪些后续反馈稳定存在，目前缺乏与此案例直接对应的官方来源。

### 来源支撑

- `BG3-PLY-002`：首轮主要来源；只能支撑“玩家案例与讨论线索”。
- `BG3-OFF-005`：直接支撑“某位 bard 会在营地临时入队”，并强调这不是稳定新同伴路线。
- `BG3-OFF-002`：除提供营地夜晚、写作流、角色反应与长休相关修复背景外，还能直接补强该事件是“Alfira 暂时入队、不是新同伴路线”的官方 framing。
- `BG3-OFF-003`：只能提供“团队愿意补量对话、保留奇怪玩法、让状态排列被覆盖”的设计意图背景，不能直接证明 bard 事件本身。

### 仍不确定的部分

- `待验证问题`：这条事件中，哪些分支与后续差异有官方或更高层来源可直接支撑？
- `待验证问题`：此案例应归为“任务与选择回流”核心案例，还是更适合作为“营地与长休循环”模块中的耦合案例？
- `待验证问题`：这条事件的稳定性与版本差异如何，是否适合在首轮正文中承担较重论证职责？

## 可写为事实的部分

- `BG3-OFF-005` 直接确认了：Dark Urge 流程中，某位 bard 会在营地暂时作为可控制角色加入。
- `BG3-OFF-002` 进一步确认了：这个 bard 就是 `Alfira`，而且该变化是暂时的，不是稳定的新同伴路线。
- 当前社区确实把该事件当作一个与长休、营地、角色后续差异相关的高辨识度案例进行讨论。
- 现有官方来源仍不能直接支撑该事件更细的内部状态细节与完整后果分布。

## 只能写为推断的部分

- 这条事件说明 BG3 会把前序选择延迟到营地节点再反馈。
- 这条事件能代表“任务系统与营地系统互相耦合”的一般规律。
- 玩家会把这条事件体验成“被世界重新解释”的高价值反馈。

## 必须保留为待验证问题的部分

- 该事件具体有哪些稳定的后续反馈。
- 这些反馈是任务脚本、角色状态还是营地事件优先级共同作用的结果。
- 这条案例是否适合继续留在首批两大案例中，还是应在后续被更稳的案例替换。

## Notes

- 这条案例目前比 Minthara 路径更弱，因为官方直接证据不足。
- 它之所以仍保留在首批案例中，是因为它能逼迫任务模块与营地/长休循环建立连接，并提醒我们不要把“任务反馈”只理解为任务日志内反馈。
- 经过 `BG3-OFF-002` 的补强后，这条案例在 `03_party_and_camp` 中更适合作为“强案例但仍需控制外推范围”的写法，而不是纯粹依赖社区讨论的悬空入口。
