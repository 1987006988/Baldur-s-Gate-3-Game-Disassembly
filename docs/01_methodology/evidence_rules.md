# Evidence Rules

## 什么是来源

来源是能被明确指认、回看和复核的材料，包括官方页面、开发者访谈、补丁说明、深度分析文章、结构化玩家案例等。

## 什么是事实

事实是被来源直接支持的陈述，例如官方明确描述的系统目标、补丁说明列出的改动、界面中可稳定观察到的行为。

写事实时:

- 必须带 Source ID
- 尽量避免价值判断
- 若事实来自玩家案例，只能陈述案例本身，不可直接泛化到全局

## 什么是推断

推断是根据多个事实、观察或案例提出的解释性判断，例如“营地循环强化了剧情节奏与资源管理的耦合”。

写推断时:

- 要说明基于哪些事实或来源
- 不把推断伪装成技术实锤

## 什么是可验证假设

可验证假设是目前证据不充分，但可以通过新增来源、案例对照或实现线索继续检验的判断。

推荐写法:

- `待验证问题：长休触发节奏是否由特定任务状态与资源条件共同控制？`

## 如何引用 source_index

- 在正文使用 Source ID，如 `BG3-OFF-002`
- 新来源先登记到 `docs/00_project/source_index.md`
- 如果来源被多个模块复用，`Where used` 需及时补充

## 如何处理互相冲突的来源

- 同时记录冲突双方
- 优先核对来源级别与时间
- 如果无法裁定，保留冲突并降级为待验证问题
- 不允许只保留更符合当前结论的一侧证据
