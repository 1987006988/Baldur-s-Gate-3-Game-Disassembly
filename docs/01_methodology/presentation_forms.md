# Presentation Forms

## 作用

本文件只定义“研究文档如何转化为展示产物”的轻量原则，不负责实际制作任何图表、卡片、时间线或幻灯片。

## 研究文档与展示产物的关系

- 研究文档是 source of truth。
- 展示产物只能压缩、重组、转述研究结论，不能替代研究文档本身。
- 如果展示内容与研究文档冲突，以研究文档为准，并回到研究文档修正。
- 没有研究文档支撑的展示结论，不应进入对外或对内展示层。

## 各模块的推荐展示形式

这里只写推荐原则，不展开制作。

- `00_core_thesis`：适合做“总命题 + 当前最强解释”的单页摘要，但必须保留结论级别与 Source ID 回溯入口。
- `01_macro_structure`：适合做“Act / 区域 / 入口 / 推进结构”示意，但重点应是结构关系，不是地图美术展示。
- `02_quests_and_choices`：适合做“选择点 - 状态变化 - 后续反馈”链路摘要，但必须能回溯到具体案例与来源。
- `03_party_and_camp`：适合做“任务结果 - 营地/长休 - 关系反馈”循环摘要，但不能把角色印象图当成系统分析。
- `04_combat_and_environment`：适合做“构筑 / 环境 / 战前布局”三类解法空间对照，但要避免沦为 build 展示。
- `05_implementation_validation`：适合做“证据来源类型与验证入口”摘要，重点是证明链而不是技术细节堆砌。
- `master_roadmap` 与状态文件：适合做阶段看板或路线摘要，但必须反映当前仓库真实阶段，而不是理想化计划。

## “研究完成”和“展示完成”的区别

- 研究完成：相关模块已有可追溯结论，区分了事实、推断、待验证问题，并已同步状态文件。
- 展示完成：已有一个面向阅读效率的压缩表达，且不歪曲研究文档中的边界、证据等级和未决问题。

研究完成先于展示完成；展示完成不代表研究已经充分完成。

## 追溯要求

- 每个展示产物都必须能回溯到对应研究文档路径。
- 每个关键判断都必须能回溯到至少一个 Source ID。
- 如果展示中做了压缩合并，仍要保留“来自哪个模块、依赖哪些来源”的说明。
- 若无法清楚回溯到研究文档与 Source ID，该展示产物视为未完成。

## 使用边界

- 本文件只提供展示原则，不引入新目录、新模板、新检查流程。
- 只有当研究文档已足够稳定时，才考虑制作展示产物。
- 如果时间有限，优先补研究文档，不优先做展示层整理。
