# Research Writing Rules

本文件只管研究写作，不管仓库级协作。

## 结论类型

- `事实`：可由来源直接支持，且必须能追溯到 `docs/00_project/source_index.md` 中的 Source ID
- `推断`：由多个事实或观察合理推出，但不是来源原句本身
- `待验证问题`：目前证据不足，但值得继续检验的假设或疑点

## 来源优先级

官方资料 > 开发者说明 > 权威深度分析 > 高质量玩家拆解 > 普通论坛讨论

## 写作约束

- 不允许把玩家感受直接写成事实
- 任何关键判断都要说明它属于事实、推断还是待验证问题
- 每个分析模块都要尽量回答:
  - 玩家看到了什么
  - 系统层发生了什么
  - 实现层如何被验证，或如何提出可验证假设
- 模块文档更新后，若结论发生变化，需要写入 `docs/00_project/decision_log.md`
- 尽量追加式编辑，避免让后续会话失去上下文

## 引用要求

- 首次使用来源时写出 Source ID
- 如果多个段落都依赖同一来源，重复标注比省略更安全
- 互相冲突的来源要并列记录，不得只保留符合当前判断的一侧
