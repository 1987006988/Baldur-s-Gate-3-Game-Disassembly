# Party and Camp

## Question

同伴、营地与长休，如何把前序选择、关系推进和延迟反馈缝合成一种“世界仍在回应玩家”的体验？

## Short answer

当前最稳的判断是：营地 / 长休不是单纯的休整容器，而是 BG3 用来集中回收前序状态变化、放大关系反馈、并把部分任务后果重新解释给玩家的折返点。对玩家来说，这会表现为“我刚才做的事在营地被重新接住”；对系统来说，这意味着任务状态、角色状态、对话可达性与 companion reaction 会在营地节点重新汇流。现阶段最适合支撑这一判断的，不是同伴剧情总整理，而是 1 条强案例、1 条候补案例和 1 组系统型补丁锚点。来源：`BG3-OFF-002`, `BG3-OFF-005`, `BG3-OFF-007`, `BG3-OFF-008`, `BG3-PLY-002`

## Current Writeback Frame

### 当前可直接写入正文的判断

- `事实`：`BG3-OFF-005` 直接确认 `Dark Urge` 流程中，某位 bard 会在营地以可控制角色身份暂时入队。这说明营地可以承接“前序触发 -> 临时角色状态变化 -> 玩家可见反馈”的链条。
- `事实`：`BG3-OFF-008` 直接确认 Minthara 的营地对话可达性、companion reaction 和关系反馈会被单独热修。这说明营地中的反应链不是可有可无的 flavour text，而是公开可见的系统边界。
- `事实`：`BG3-OFF-002` 与 `BG3-OFF-007` 共同表明，camp night、character reaction、writing / flow、跨 Act 反思时刻和营地队伍管理都在被持续维护。这说明营地不仅是休息界面，也是状态汇流与反馈重放节点。
- `推断`：BG3 的一部分高反应性来自“延迟反馈被放到营地 / 长休节点重新呈现”，而不只来自任务现场的即时结算。
- `推断`：营地最重要的系统价值之一，不是单独提供对白，而是把分散在任务、关系和角色状态中的结果重组为玩家能感到的二次反馈。

### 当前不应直接写成正文事实的判断

- `待验证问题`：`Dark Urge bard` 事件具体有哪些稳定后续分支、哪些后续差异在多个版本中都成立，目前还缺乏与之逐条对应的更高层官方来源。
- `待验证问题`：`Minthara` 案例更像角色特例修补，还是足以代表更普遍的营地反馈调度规律，目前仍不能直接下定论。
- `待验证问题`：长休频率、营地事件优先级与任务状态之间是否存在更稳定的公开可验证模式，目前证据还不够。

## Player-layer observations

- 玩家会把营地理解成“刚才发生的事开始产生后续意义”的地方，而不只是恢复资源的地方。
- 某些高辨识度事件之所以让玩家记忆深刻，不是因为任务现场立即结算，而是因为营地 / 长休把前序行为重新转化成角色存在差异、关系评论或后续对话。
- 当营地中的对话、同伴评论或角色可达性缺失时，玩家会直接感到“后果没有被接住”。来源：`BG3-OFF-005`, `BG3-OFF-008`, `BG3-PLY-002`

## System-layer explanation

- 营地 / 长休把任务状态、角色状态、队伍状态和关系反馈集中到一个低压节点里重新触发，因此能放大反馈感。
- 这类反馈不一定改变任务日志本身，却会改变玩家对前序行为的解释方式：任务后果不再只存在于任务现场，也会折返到营地中的角色状态与关系层。
- 从当前证据看，营地更像“反馈汇流点”而不是“同伴剧情列表”；它既承接前序结果，也承接后续关系推进和对话可达性维护。来源：`BG3-OFF-002`, `BG3-OFF-005`, `BG3-OFF-007`, `BG3-OFF-008`

## Implementation-layer evidence / hypothesis

### 强案例：Dark Urge 营地 bard 事件

- `事实`：`BG3-OFF-005` 直接确认了营地中的临时入队状态变化。
- `事实`：`BG3-PLY-002` 只能作为玩家侧案例线索，用来观察玩家如何感知“长休触发 -> 角色替换 / 保留 -> 后续差异”，不能单独承担系统事实。
- `推断`：这条案例最适合作为首批强案例，不是因为它证据最完备，而是因为它最清楚地闭合了“前置触发 -> 营地状态变化 -> 后续差异感知”这一条链。
- `待验证问题`：它适合承载多大论证权重，仍取决于后续是否能补到更直接的后续反馈来源。来源：`BG3-OFF-005`, `BG3-PLY-002`, `BG3-OFF-002`

### 候补案例：Minthara 营地反应链

- `事实`：`BG3-OFF-008` 直接暴露了营地中的对话可达性、同伴评论与关系反馈会被单独维护。
- `事实`：`BG3-OFF-002` 提供了更高层的补丁背景，说明 camp night、character reaction 与 writing / flow 是持续修复的对象。
- `推断`：这条案例更适合作为候补案例或系统边界暴露点，而不是首批正文里的主论据，因为它强于“营地反馈链存在”，弱于“完整剧情回流闭环”。
- `待验证问题`：是否能找到非 Minthara 角色的平行条目，证明这不是单角色历史修复。来源：`BG3-OFF-008`, `BG3-OFF-002`

### 系统型补丁锚点

- `事实`：`BG3-OFF-002` 直接暴露 camp night、character reaction、writing / flow 与任务后续反应的维护面。
- `事实`：`BG3-OFF-007` 直接暴露营地既承担队伍状态处理，也承担跨 Act 反思时刻。
- `推断`：这些补丁条目共同说明，营地 / 长休不是个别剧情特例的落点，而是一个被长期维护的反馈层。来源：`BG3-OFF-002`, `BG3-OFF-007`
