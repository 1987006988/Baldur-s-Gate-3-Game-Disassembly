# Core Thesis

## Question

《博德之门 3》为什么会让玩家持续感到“我能自己找办法，而且世界真的会接住这些办法”？

## Short answer

当前最强解释版本是：BG3 的高反应性不是单靠分支数量堆出来的，而是把多入口空间、规则驱动的解法、任务与营地中的状态回流，以及电影化反馈包装缝合到同一条体验链里。玩家先在局部场景里获得“我能这样试”的能动性，再在后续区域、营地或角色反馈里获得“系统真的记住了”的确认。

## Facts

- `事实`：官方 About 页把 BG3 定义为一个强调选择后果、电影化叙事、队伍协作与基于 D&D 5e 规则战斗的 RPG，而不是单独强调某一条分支系统。来源：`BG3-OFF-001`
- `事实`：官方 About 页用“影咒森林、幽暗地域洞穴、博德之门城市”等差异化区域来描述游戏空间，说明官方对宏观推进的表达是多区域串联，而非单一大地图。来源：`BG3-OFF-001`
- `事实`：Adam Smith 在官方 Xbox Podcast 中把 BG3 的关卡与路径设计比作 “Swiss cheese”，强调场景有很多洞、很多进入方式，并明确把这种结构和玩家的非常规解法联系起来。来源：`BG3-OFF-003`
- `事实`：同一访谈中，Adam Smith 还提到团队在上线后补了大量额外对话与过场，以覆盖玩家做出的更多状态排列与结果路径。来源：`BG3-OFF-003`

## Inferences

- `推断`：BG3 的自由感不是“选项多”本身，而是“多入口尝试”与“延迟反馈承接”同时存在。前者主要由空间与规则系统提供，后者主要由任务、营地、角色状态和补充对白承担。来源：`BG3-OFF-001`, `BG3-OFF-003`
- `推断`：电影化叙事在这里更像是反馈放大器，而不是高反应性的唯一来源。真正让玩家觉得世界有回应的，是系统先允许偏离标准路径，再在后续环节把偏离写回内容层。来源：`BG3-OFF-003`
- `推断`：仓库当前已经形成的三个局部模块方向可以被统一到同一解释框架下：
  - `02_quests_and_choices.md` 负责解释状态回流如何被玩家感知
  - `03_party_and_camp.md` 负责解释营地 / 长休为什么是反馈折返点
  - `04_combat_and_environment.md` 负责解释“选择”为什么不只发生在对白里。来源：`BG3-OFF-001`, `BG3-OFF-003`

## What This Thesis Does Not Yet Claim

- `待验证问题`：目前还不能把“所有重要选择都会被充分回流”写成事实；现有更稳的结论是，Larian 把“覆盖更多可预期外路径”当成明确设计方向，但覆盖范围和稳定度仍有边界。来源：`BG3-OFF-003`
- `待验证问题`：目前也还不能把“高反应性”完全等同于“任务系统先进”；现有证据更支持它是一种跨任务、营地、战斗与区域结构的组合效果。来源：`BG3-OFF-001`, `BG3-OFF-003`
