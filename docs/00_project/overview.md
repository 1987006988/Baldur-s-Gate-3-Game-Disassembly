# 项目总览

## 总命题

《博德之门3》如何把桌面规则、电影化叙事和系统性玩家能动性缝合成一个高反应性的 CRPG。

## 研究范围

- 玩家为何感到“我真的可以这样做，而且世界会回应我”
- 任务、选择、成长、战斗、探索、长休、同伴关系如何形成互相牵引的系统网络
- 官方公开资料、开发者表述、UI 与可观察结构如何为上述判断提供验证线索

## 不做什么

- 不按章节完整复述剧情
- 不追求全收集式资料归档
- 不把每个职业、每条支线都拆成独立百科条目
- 不在缺证据时强行下技术实现结论

## 为什么三层拆解更有效

普通剧情整理容易停在“发生了什么”，功能罗列容易停在“有什么系统”。三层拆解要求把玩家感受、系统耦合和实现证据放在同一张工作台上，迫使研究回答三个更有用的问题:

1. 玩家具体在何处感到自由、沉浸、被反馈。
2. 这些感受背后有哪些规则循环与资源约束。
3. 哪些判断已有证据，哪些还只是合理猜测。

这能让仓库持续产出可修正、可接力、可验证的研究结论，而不是一次性观后感。
