# Repository Collaboration Rules

本仓库用于《博德之门3》长期研究拆解，边界是可验证研究与上下文维护，不是随手笔记区，也不是通用知识库。

## 导航

- 先读 `README.md`
- 再读 `docs/00_project/current_state.md`
- 再读 `docs/00_project/next_step.md`
- 再读 `docs/00_project/source_index.md`
- 研究写作规则见 `docs/AGENTS.md`
- 大型任务规划规则见 `.agent/PLANS.md`

## 执行要求

- 修改任何研究文档前，必须先阅读 `README.md`、`docs/00_project/current_state.md`、`docs/00_project/next_step.md`、`docs/00_project/source_index.md`
- 完成任何有意义的任务后，必须同步更新:
  - `docs/00_project/current_state.md`
  - `docs/00_project/next_step.md`
  - `docs/00_project/decision_log.md`（如有决策变化）
  - `docs/00_project/source_index.md`（如引入新来源）
- 复杂任务或跨度较大的结构调整，必须先使用 ExecPlan
- 不允许把项目知识大量塞进 `AGENTS.md`；这里只写协作规则

## Done 定义

- 文档有实质更新
- 状态文件已同步
- 检查命令通过
