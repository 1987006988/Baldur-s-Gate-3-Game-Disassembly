# ExecPlan: Party, Camp, and Long-Rest Feedback Loops

## Goal

为 `docs/03_analysis/03_party_and_camp.md` 建立第一份可执行研究计划，明确这个模块如何从“营地 / 长休是任务反馈折返点”的当前判断，推进到可追溯、可接力、可继续扩展的案例驱动正文。

本计划要回答的核心问题是：
> BG3 如何把同伴、营地、长休与任务后果缝合成一个既像叙事休整点、又像系统反馈汇流点的循环？

## Why it matters

- 任务模块已经证明，BG3 的高反应性并不只发生在任务现场，营地 / 长休会承担二次反馈与再解释功能。
- 如果不把营地模块独立出来，后续研究很容易把营地只当成“任务反馈附属页”，而不是一个有自己节奏、入口和验证问题的系统。
- 这个模块天然连接任务、同伴关系、长休资源循环与实现验证，适合作为第二份 ExecPlan，把项目从单模块突破推进到跨模块协同。

## Scope

本计划只覆盖以下范围：

- 定义营地 / 长休模块的研究边界
- 筛出首批适合本模块的来源类型
- 选出 2 个以内的首批案例方向
- 建立“任务后果 -> 营地 / 长休触发 -> 同伴或状态反馈”的最小证据链模板
- 标出与任务模块、实现验证模块的最小必要耦合点

本计划明确不覆盖：

- 全面重写 `03_party_and_camp.md`
- 全量梳理所有同伴个人剧情
- 大规模补社区资料
- 展示层产物制作
- 对恋爱 / 好感度系统做泛化总结

## Milestone 1 boundary decisions

### 本模块优先回答什么

- 营地 / 长休如何把前序任务、角色状态与关系反馈汇总成玩家可感知的二次反馈节点
- 哪些反馈更像“任务后果在营地被重新解释”
- 哪些反馈更像“营地自身在调度关系推进与事件节奏”
- 玩家为什么会把营地感受到的内容理解成“世界在继续回应我”

### 本模块当前不优先回答什么

- 单个同伴从 Act I 到 Act III 的完整人物弧线
- 恋爱系统、好感度系统或 companion approval 的全貌整理
- 全量 camp scene / long rest scene 列表
- 任何缺乏来源支撑的“最佳长休顺序”经验帖总结

### 长休 / 营地 / 同伴反馈的最小关系图

1. 前置触发：玩家在任务、区域推进、战斗结局或关系互动中造成状态变化。
2. 节奏切换：长休或回营地把这些分散状态带入可集中触发的节点。
3. 反馈落点：营地以对话、临时角色状态、同伴反应、事件插入或后果再解释的形式回收这些状态。
4. 玩家体验：玩家把这种延迟且重组后的反馈理解成“同伴活着”以及“世界还记得我刚才做了什么”。

### 问题归属边界

- 继续留在 `02_quests_and_choices` 的问题：
  - 某条任务路径本身如何记录状态差异
  - 某个决定是否跨区域 / 跨章节回流
  - 任务系统是否允许非标准解决方式
- 转交给 `03_party_and_camp` 的问题：
  - 营地如何承接这些状态差异并把它们重组为可感知反馈
  - 长休如何影响反馈节奏和事件显示层
  - 同伴反应何时更像“反馈放大器”而不是独立支线
- 暂留给 `05_implementation_validation` 的问题：
  - 哪些公开来源足以支撑“营地反馈是稳定机制”这一判断
  - 哪些补丁 / 热修条目可作为状态边界验证入口

### 首批案例准入标准

- 至少能说清一个前置触发，不只是“营地里发生了一段剧情”
- 至少能说清反馈发生在营地 / 长休节点，而不是任务现场即时结算
- 至少能区分反馈主要属于任务后果、角色状态，还是关系推进
- 至少有 1 条官方或开发者来源作为事实锚点；社区来源最多只作案例线索
- 能明确写出事实、推断与待验证问题，避免把玩家感受直接写成系统事实

### 当前最可能的首批案例方向

- 强候选：`Dark Urge 营地 bard 事件`
  - 理由：已经有 `BG3-OFF-005` 作为直接官方入口，且天然体现“任务后果折返到营地”
- 候补方向：`Patch / Hotfix 中与 camp night、character reaction、writing and flow 相关的修正串`
  - 理由：它未必是单一剧情案例，但很适合作为“营地是反馈汇流点”的系统证据链入口

## Working assumptions

- `推断`：营地最重要的系统价值，不是单独提供对话，而是把分散在任务、关系、资源消耗中的状态变化汇总成玩家可感知节点。
- `推断`：长休既是资源重置点，也是反馈节奏开关；因此它会显著影响玩家对“世界在回应我”的主观强度。
- `待验证问题`：营地反馈究竟更常是任务状态的延迟显示层，还是本身就承担独立的关系推进与事件调度逻辑？
